Detailed Description

Contains the API related to hints (configuration variables).

Refer to the official SDL2 wiki or the SDL_hints.hpp header for details regarding any specific hint type.

Adds a callback to observe changes of the value of the specified hint.

A callback handle object is returned, which can be used to easily disconnect the callback later. This function can be used with any function object that is stateless, such as traditional function pointers and lambdas. The simplest way to add a callback is with a lambda and no explicit user data.

Note
The callback will be immediately invoked with the current value of the hint.

Adds a callback to observe changes of the value of the specified hint.

This function returns a callback handle object, which can be used to easily disconnect the callback at a later time.

The signature of the callable should be equivalent to void(UserData*, str, Hint::value_type, Hint::value_type).

Note
This function can be used with any function object that is stateless, such as traditional function pointers or non-capturing lambdas.

The following is an example of how usage of this function might look.

auto callable = [](int* data,
cen::str name,
cen::hint::render_driver::value_type previous,
cen::hint::render_driver::value_type current) {
// Do stuff when the value of the hint is updated...
};
int foo = 42;
auto handle = cen::add_hint_callback_ex<render_driver>(callable, &foo);

◆ set_hint()

template<typename Hint , hint_priority priority = hint_priority::normal, typename Value , detail::enable_if_hint_arg_t< Hint, Value > = 0>
auto cen::set_hint ( const Value &  value) -> result

Sets the value of the specified hint.

This function will only accept values that are related to the specified hint, supplying the wrong kind of value causes a compile-time error. See the related hint class for more details about the associated value type. However, whilst this function is type-safe, it doesn't ensure that correct values are specified for the hints.
